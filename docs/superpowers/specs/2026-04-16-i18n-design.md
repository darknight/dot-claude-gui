# 国际化（i18n）完善设计

**日期：** 2026-04-16
**范围：** 全量覆盖（Option C）
**目标语言：** zh-CN、en-US、ja-JP（主力三语）+ es-ES、fr-FR、ko-KR（扩展位）
**方案：** 自研，沿用现有 `t()` 函数签名，零依赖

---

## 1. 背景与现状

`src/lib/i18n.ts` 目前是一个最小实现：

- 类型写死为 `"zh-CN" | "en-US"`。
- 词条直接内联在文件里（当前只有 2 条）。
- 全仓库只有 `src/lib/components/memory/MemoryList.svelte` 一个文件真正调用了 `t()`。
- 6 个 Svelte 文件合计 28 处硬编码中文（含设置页的"语言"label 自身），以及零散的硬编码英文片段。
- `AppSettingsView.svelte` 语言下拉只列出 zh-CN 与 en-US，默认 `zh-CN` 对非中文母语用户冷启动体验差。
- CLAUDE.md 已明令 "Do NOT hardcode Chinese or English text in components"，但未落地。

此设计把 i18n 架构扩到三主力语言，收敛所有硬编码文本，并为未来 es/fr/ko 预留扩展位。

## 2. 目标

1. 支持 zh-CN、en-US、ja-JP 三种主力语言，UI 全量通过 `t()` 渲染，零硬编码中/英/日字面量出现在业务组件中。
2. 保留 es-ES、fr-FR、ko-KR 的扩展位：结构就绪、词条先留空、回退到 en-US，后续只需补翻即可启用。
3. 首次启动按系统语言自动探测 locale，既有用户选择永远被尊重。
4. 新增词条时通过 TS 类型强制同步到所有语言包，防止漏翻。
5. 语言切换在 UI 中实时生效，无页面刷新，无首屏闪烁。
6. 不引入第三方 i18n 库，遵守 "Svelte 5 runes only" 的既有约束。

## 3. 非目标（YAGNI）

- 不支持 ICU 复数/性别规则。词条设计阶段改写为无需复数的句式（例如 "N items" 而非 "1 file / 2 files"）。
- 不做日期/数字本地化。当前 UI 不展示对语言敏感的日期/数字格式。
- 不做翻译热加载。桌面应用用户不会在运行时修改翻译文件。
- 不引入 Vitest 或 e2e 测试。遵循项目"前端无测试套件"的既定约定。
- 不支持 RTL 语言。不在支持列表内。

## 4. 架构

### 4.1 文件布局

```
src/lib/
├── i18n.ts                          # 扩展后的 t() + 类型 + 首启探测
└── locales/
    ├── zh-CN.json                   # 简体中文 — key 全集权威
    ├── en-US.json                   # English — 兜底语言
    ├── ja-JP.json                   # 日本語
    ├── es-ES.json                   # Español（扩展位，值为空字符串）
    ├── fr-FR.json                   # Français（扩展位，值为空字符串）
    └── ko-KR.json                   # 한국어（扩展位，值为空字符串）
```

### 4.2 Locale 类型

```ts
export const SUPPORTED_LOCALES = [
  "zh-CN", "en-US", "ja-JP",
  "es-ES", "fr-FR", "ko-KR",
] as const;
export type Locale = (typeof SUPPORTED_LOCALES)[number];

// 下拉只暴露主力三语；扩展位未翻译完成前不进 UI
export const ACTIVE_LOCALES: Locale[] = ["zh-CN", "en-US", "ja-JP"];
```

### 4.3 词条表类型模型

`zh-CN.json` 作为 key 全集权威。通过 `tsconfig.json` 的 `resolveJsonModule` + `as const` 风格的类型推断（或 `typeof zhCN` 做类型索引），所有其他语言包的类型被收窄为 `Record<keyof typeof zhCN, string>`。效果：

- 新增 key 只加到 zh-CN.json 而忘了同步其他 5 语 → `pnpm build` 的 `tsc --noEmit` 阶段报错 `Property 'X' is missing in type ...`。
- 扩展位（es/fr/ko）的值允许是空字符串，运行时走 en-US 兜底。

### 4.4 key 命名规则

扁平 camelCase 点分：`<module>.<description>`。与现有 2 条 key 无缝衔接。

## 5. 模块与职责

### 5.1 `src/lib/i18n.ts`（重写）

```ts
import { appSettingsStore } from "$lib/stores/appsettings.svelte";
import zhCN from "./locales/zh-CN.json";
import enUS from "./locales/en-US.json";
import jaJP from "./locales/ja-JP.json";
import esES from "./locales/es-ES.json";
import frFR from "./locales/fr-FR.json";
import koKR from "./locales/ko-KR.json";

export const SUPPORTED_LOCALES = [
  "zh-CN", "en-US", "ja-JP", "es-ES", "fr-FR", "ko-KR",
] as const;
export type Locale = (typeof SUPPORTED_LOCALES)[number];
export const ACTIVE_LOCALES: Locale[] = ["zh-CN", "en-US", "ja-JP"];

export type MessageKey = keyof typeof zhCN;

const bundles: Record<Locale, Record<MessageKey, string>> = {
  "zh-CN": zhCN,
  "en-US": enUS,
  "ja-JP": jaJP,
  "es-ES": esES,
  "fr-FR": frFR,
  "ko-KR": koKR,
};

export function t(key: MessageKey, params?: Record<string, string | number>): string {
  const lang = (appSettingsStore.preferences.language as Locale) ?? "en-US";
  let text = bundles[lang]?.[key] || bundles["en-US"][key];
  if (!text) {
    if (import.meta.env.DEV) console.warn(`[i18n] missing key: ${key}`);
    return key;
  }
  if (params) {
    for (const [k, v] of Object.entries(params)) {
      text = text.replace(`{${k}}`, String(v));
    }
  }
  return text;
}

export function detectInitialLocale(): Locale {
  const nav = typeof navigator !== "undefined" ? navigator.language : "";
  if (nav.startsWith("zh")) return "zh-CN";
  if (nav.startsWith("ja")) return "ja-JP";
  return "en-US";
}

export function localeDisplayName(loc: Locale): string {
  return {
    "zh-CN": "简体中文",
    "en-US": "English",
    "ja-JP": "日本語",
    "es-ES": "Español",
    "fr-FR": "Français",
    "ko-KR": "한국어",
  }[loc];
}

export function isSupportedLocale(s: unknown): s is Locale {
  return typeof s === "string" && (SUPPORTED_LOCALES as readonly string[]).includes(s);
}
```

### 5.2 `src/lib/stores/appsettings.svelte.ts`（修改）

**变更点：**

1. `AppConfig.language` 字段类型从 `string` 改为 `Locale | undefined`（向后兼容 config.json 里没有该字段的情况）。
2. 默认值从硬编码 `"zh-CN"` 改为 `undefined`。
3. `load()` 末尾新增：
   - 若 `preferences.language` 未定义或不在 `SUPPORTED_LOCALES`（脏数据自愈），调用 `detectInitialLocale()` 并 `save()`。
4. 新增 `loaded = $state(false)` 字段，`load()` 返回前置 `true`，用于首屏 gating。

**伪代码：**

```ts
async load() {
  try {
    const json = await invoke<string>("read_app_config");
    const saved: Partial<AppConfig> = JSON.parse(json);
    this.preferences = { ...this.preferences, ...saved };
  } catch {}

  // localStorage 迁移（沿用现有逻辑）
  // ...

  // 脏数据自愈 + 首启探测
  if (!isSupportedLocale(this.preferences.language)) {
    this.preferences.language = detectInitialLocale();
    await this.save();
  }

  this.loaded = true;
}
```

### 5.3 `src/lib/components/appsettings/AppSettingsView.svelte`（修改）

- `<option>` 硬编码改为 `{#each ACTIVE_LOCALES as loc}<option value={loc}>{localeDisplayName(loc)}</option>{/each}`。
- 该文件内现有 8 处中文硬编码（含"语言"label）全部改走 `t("appsettings.*")`。

### 5.4 其他包含硬编码文本的 Svelte 文件

逐文件抽取到对应模块前缀：

- `src/App.svelte`（13 处） → `nav.*`、`common.*`。
- `src/lib/components/skills/SkillList.svelte`（2 处） → `skills.*`。
- `src/lib/components/claudemd/ClaudeMdEditor.svelte`（1 处） → `claudemd.*`。
- `src/lib/components/claudemd/ClaudeMdList.svelte`（2 处） → `claudemd.*`。

实现阶段需要对所有组件做一遍搜索，捕获 grep 遗漏的英文硬编码（如按钮文本 "Save"、"Cancel"、"Loading..." 等）。

### 5.5 `src/App.svelte` — 首屏 gating 与 `document.lang`

**首屏 gating：**

```svelte
{#if appSettingsStore.loaded}
  <!-- 既有主界面 -->
{:else}
  <!-- 空的占位或 loading spinner，避免用默认 language 渲染一帧 -->
{/if}
```

**`document.lang` 同步：**

```svelte
$effect(() => {
  const lang = appSettingsStore.preferences.language;
  if (lang) document.documentElement.lang = lang;
});
```

## 6. 数据流场景

### 6.1 冷启动（全新用户）

1. `App.svelte` `onMount` → `appSettingsStore.load()`。
2. `invoke("read_app_config")` 返回空对象或无 `language` 字段。
3. `preferences.language` 仍为 `undefined`。
4. `load()` 末尾自愈逻辑：`detectInitialLocale()` 按 `navigator.language` 返回 zh-CN / ja-JP / en-US，写入 `preferences.language` 并 `save()` 到 `~/.dot-claude-gui/config.json`。
5. `loaded = true` → 主界面渲染，所有 `{t("...")}` 使用探测后的值，无闪烁。

### 6.2 热启动（已有配置）

1. `load()` 读到 `config.json` 里 `language = "zh-CN"` 等合法值。
2. 自愈校验通过（在 `SUPPORTED_LOCALES` 中），不覆盖。
3. 用户既有选择被尊重。

### 6.3 运行时切换语言

1. 用户在 `AppSettingsView` 切 `<select>`。
2. `onchange` → `appSettingsStore.update({ language: "ja-JP" })`。
3. `this.preferences = { ...this.preferences, language: "ja-JP" }`（`$state` 赋值触发响应式传播）。
4. `save()` 写入 `config.json`。
5. Svelte 5 响应式：所有模板里 `{t("...")}` 自动重执行，UI 切到日文。
6. `$effect` 同步 `document.documentElement.lang = "ja-JP"`。

### 6.4 key 缺失回退

```
t("foo.bar") 当 lang = ja-JP，且 ja-JP.json 里 foo.bar = ""
  → bundles["ja-JP"]["foo.bar"] = "" (falsy)
  → || bundles["en-US"]["foo.bar"] = "Foo Bar"
  → 返回 "Foo Bar"

若 en-US 也空 / undefined
  → 返回 "foo.bar" 字面量
  → dev 环境 console.warn
```

### 6.5 Loader 时序保证

所有 6 个 JSON 通过顶层静态 `import` 由 Vite 打包成同步模块，运行时没有异步等待。`t()` 第一次被调用时 bundles 已就绪。唯一需要等待的异步资源是 `appSettingsStore.preferences.language` 本身，由 `loaded` 标志位做首屏 gating 保证。

## 7. 错误处理

| # | 场景 | 策略 |
|---|---|---|
| 1 | JSON 文件解析失败（打包期） | Vite 构建直接失败，编译期错误 |
| 2 | config.json 里 language 是未知字符串 | `load()` 自愈：重置为 `detectInitialLocale()` 并 save |
| 3 | key 在所有语言都缺失 | 返回 key 字面量 + dev `console.warn` |
| 4 | params 漏传占位符 | `{name}` 原样保留，不抛异常 |
| 5 | params 多传未使用的 key | 忽略，无副作用 |
| 6 | 扩展位 locale 词条全为空字符串 | 自动走 en-US 兜底（空字符串是 falsy） |
| 7 | `navigator.language` 为空或异常 | `detectInitialLocale()` 默认返回 en-US |
| 8 | localStorage 旧值里 language 非法 | 经过场景 2 的自愈逻辑纠正 |

## 8. 测试策略

**前端无测试套件是项目既定约定（CLAUDE.md）。本设计不引入 Vitest。**

**第一道防线：TS 类型即测试。**

- zh-CN.json 结构权威 → 其他 5 语 bundle 类型收窄为 `Record<keyof typeof zhCN, string>`。
- 加新 key 忘了同步其他语言 → `tsc --noEmit` 报错。
- 组件里 `t("typo.key")` 用了不存在的 key → 联合类型不包含该字符串 → TS 报错。

**第二道防线：实现完成后手动冒烟清单。**

1. `rm ~/.dot-claude-gui/config.json` 后启动 → 根据 `navigator.language` 自动选中正确语言。
2. 设置页依次切 zh-CN / en-US / ja-JP → 所有 9 个导航模块 UI 立即切换，无闪烁。
3. 手改 `config.json` 写 `"language": "xx-YY"` → 启动后被自愈为探测值。
4. dev 环境故意删 en-US.json 里某个 key → `console.warn` 出现。
5. `document.documentElement.lang` 随语言切换变化（DevTools 查看 `<html>`）。
6. 关闭再启动 → 上次选择的语言被保持。

**第三道防线：视觉巡检。**

启动后走遍 9 个导航模块，用 DevTools Console 搜 `[i18n] missing key` 警告，捕捉漏翻。

## 9. 翻译内容交付

- 所有 zh-CN / en-US / ja-JP 词条由 Claude 在实现阶段一次性给出。
- zh-CN 由用户审校（母语直接改）。
- en-US 由用户扫读，有违和处由 Claude 修正。
- ja-JP 由用户后续交给母语者或翻译工具复核；若术语不准，用户在审校时指出，Claude 重改。
- es-ES / fr-FR / ko-KR 的 JSON 文件中所有 key 的值先留空字符串；后续补翻不需要改代码。
